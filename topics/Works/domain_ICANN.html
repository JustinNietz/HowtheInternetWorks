Domain names and ICANN

What is ICANN?

ICANN (The Internet Corporation for Assigned Names and Numbers) is a not-for-profit partnership of people all across the world dedicated to keeping the internet secure and stable.

Formed in 1998, ICANN coordinates unique identifiers to allow computers to know where to find each other. These unique identifiers concern the Internet’s global Domain Name System (DNS).

ICANN is made up different groups, each represent a different interest on the Internet and all of which contribute to any final decisions that ICANN’s makes.

There are three “supporting organizations” that represent:

  • The organizations that deal with IP addresses
  • The organizations that deal with domain names
  • The managers of country code top-level domains (a special exception as explained at the bottom).

ICANN's logo, formed in 1998

What is a Domain Name?

A Domain Name - such as icann.org – is essentially the address of a person or an organization on the internet. They are formed by the rules and procedures set by the Domain Name System (DNS). Any name registered in the DNS considered a domain name.

They are comprised of two elements: before and after “the dot”:

  • Before “the dot” – This is the actual domain name that you register when you want to create a website. These are sold by a large number of registrars which charge whatever they want.
  • After “the dot” – endings such as “com, “net”, and “org” are known as a top-level domain (TLD). One company (registry) in each case is in charge of all domains ending with one of these TLDs. This registry has a full list of domains directly under that name as well as all of the IP addresses which those names are associated with.

Why do we need ICANN and domain names?

Through ICANN’s coordination role of the Internet’s naming system, it does have an important impact on the expansion and evolution of the Internet. This system organizes all the websites on the web. Without this system, there would be no way to categorize all the website names.

The organization makes sure there are never two domain or IP addresses that are the same throughout the entire internet. ICANN does not run the entire system. It only helps co-ordinate how IP and domain names are supplied in order to avoid repetition or clashes.

Coined as the term universal resolvability, ICANN guarantees that wherever you are on the network – and hence the world – that you receive the same predictable results when you access the network. Without this, you could end up with an Internet that worked entirely differently depending on your location on the globe.
