DNS

Definition

A DNS (domain name system) is a system that associates names like www.example.com with the corresponding addresses. Your computer uses the DNS to look up domain names and get the associated IP address which is used to connect your computer to the destination on the internet.

One DNS server can handle all the requests from all devices. DNS servers are connected In a Distributed Hierarchy. They split up responsibility for the major domains such as .org, .com, .next, etc.

DNS was created to be an open and public communication protocol for government and educational institutions. Because of its openness, however, DNS is susceptible to cyber attacks.

An example attack is called DNS spoofing when a hacker taps into a DNS server and changes it to match a domain name with the wrong IP address. They send the user to an imposter website. To prevent this, a secure link between your computer and a website must be established using HTTPS protocols. This topic is discussed further on the next page.

Why we use DNS

DNS is designed to make the internet accessible to human beings. The main way computers find one another is through their IP Address. Because it is difficult for the human mind to remember long lists of numbers, DNS uses letters rather than numbers and then links a precise series of letters with a precise series of numbers.

For example, instead of typing 192.0.32.7 into your web browser, you can just type in icann.org. There are many advantages to this system. One advantage is that a particular domain name does not have to be tied to one particular computer since the link between a certain domain and IP address can be changed quickly and easily.

Because of the constantly updating DNS infrastructure, this change will be recognized by the entire Internet within 48 hours.
