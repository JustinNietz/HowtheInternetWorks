HTTP and HTTPS protocols

The Internet is a design philosophy expressed in a set of protocols. When we talk about a protocol, we think of a well-known set of rules and standards that are used for communication between machines. If all parties agree on using the same protocol, then this agreement will allow them all to communicate without any trouble. Furthermore, this design helps to adopt new technologies in the future.

HTTP Protocol

How could all computers in the whole world access the Internet and communicate with each other? Let us go through the steps. First, we need to choose a web browser and open it. Our next task is to type a web address or URL (Uniform Resource Locator) of the website we would like to visit. After giving the proper web address, our computer gets in touch with a server in just a couple of milliseconds. Next, your computer gets a response from the server in HTTP (Hyper Text Transfer Protocol) language. This language is used for communication between web browsers and servers. The conversation between our computer and the server are mainly “GET” requests. In this type of request, we are asking simply for a specific document written in HTML (Hyper Text Markup Language). The text on a website is already included in the HTML, however, other elements like images or videos are separate files with their unique URLs. Each element needs a “GET” request to appear properly on the webpage. If there are too many “GET” requests at once, we experience a website that loads extremely slowly.

A “GET” request looks like this.
GET /hello.htm HTTP/1.1
User-Agent: Mozilla/4.0 (compatible; MSIE5.01; Windows NT)
Host: www.tutorialspoint.com
Accept-Language: en-us
Accept-Encoding: gzip, deflate
Connection: Keep-Alive

Occasionally, we send information, too, such as filling out a form or typing keywords into a search bar. These actions require a “POST” request to the server. Let us look at an example. We would like to log into our Spotify account to listen to some music. To do that, we need to give our email address and password. This means that our computer or smartphone needs to send a “POST” request with the given information to Spotify’s server. If we sent the correct data for login, the server would let us into our Spotify account successfully. But during the response, our computer gets a cookie which will help us log in the next time without this whole process. This cookie acts as an ID card that will identity us immediately the next time we login.

This is a "POST" request.
POST /cgi-bin/process.cgi HTTP/1.1

Host: www.tutorialspoint.com

Content-Length: length

Securing Our Data

The Internet is open for anyone, and its connections are all shared. Furthermore, information is sent in plain text. These traits lead to the fact that hackers can look and get our private data. These cybercriminals have different methods to get our data which infect our computers with viruses, tricking us into phishing scams, or using our computer for DDOS attacks in cyberspace.

How can we prevent these things from happening? We need to secure the communication channels on the Internet. To do this, we must check the website we want to visit whether it has SSL (Secure Sockets Layer) and TLS (Transport Layer Security) certificates. These certificates ensure that the channel between the computer and the server is secure so our computer is well-protected during browsing on this website. When we see a small lock next to HTTPS – added S for Secure – in the browser address bar, we can be sure that this website is verified and not dangerous for us.

Encryption

The Internet is a public space. However, we frequently use it for sharing private data like credit card information or passwords to our accounts on a daily basis. How do we keep them secret? The solution to this is encryption. Using encryption, we scramble or change our original message. After hiding our data from prying eyes, the receiver of our message needs to decrypt it. With this process, the message becomes readable again. To decrypt a message, we need a key – a password for unlocking the message. Nowadays, we use 256-bit keys which are impossible for humans and computer to crack.

There are two types of encryption. First, is Symmetric Encryption wherein the sender and receiver use the same key to encrypt and decrypt a message. This encryption is used by two people who can agree on the key in private before using it, but not for communication on the Internet.

Computers use a different method: Asymmetric encryption. This method involves using two different keys. There is a public key that can be shared with anybody, and there is a private key that belongs to the receiver only. With the public key, we can encrypt a message and send this to the receiver. Anyone has access to this key so anyone can send a message. On the other hand, getting the message is limited only to the computer that has the private key. When we see the lock in front of a web address, we can be sure that this website uses public-key cryptography; the messages between parties are properly encrypted on this web page.

The Need for Security Grows

Some tendencies show the need for a secure Internet intensifies. More and more people start to browse on the Internet, and more and more activities are taking place on the Internet. This means that there will be exponentially more private data on the Internet that needs protection.
