HTTP and HTTPS protocols

The Internet is a design philosophy expressed in a set of protocols. When we talk about a protocol, we think of a well-known set of rules and standards that are used for communication between machines. If all parties agree on using the same protocol, then this agreement will allow them all to communicate without any trouble. Furthermore, this design helps to adopt new technologies in the future.

HTTP Protocol

How could all computers in the whole world access the Internet and communicate with each other? Let us go through the steps. Firstly, we need to choose a web browser and open it. Our next task to type a web address or URL (Uniform Resource Locator) of the website we would like to visit. After giving the proper web address, our computer gets in touch with a so-called server; in just a couple of milliseconds, our computer gets a response from the server in HTTP (Hyper Text Transfer Protocol) language. This language is used for communication between web browsers and servers. The conversation between our computer and the server are mainly “GET” requests. In this type of request, we are asking simply for a specific document written in HTML (Hyper Text Markup Language). The text on a website is already included in the HTML, however other elements like images or videos are separate files with their unique URLs. Each element needs a “GET” request to appear properly on the webpage. If there are too many “GET” requests at once, we can experience that the website loads extremely slowly.

A “GET” request looks like this.
GET /hello.htm HTTP/1.1
User-Agent: Mozilla/4.0 (compatible; MSIE5.01; Windows NT)
Host: www.tutorialspoint.com
Accept-Language: en-us
Accept-Encoding: gzip, deflate
Connection: Keep-Alive

Occasionally, we send information too, for example, when we fill out a form or type a keyword into the search bar. These actions require a “POST” request to the server. Let us look at an example. We would like to log in to our Spotify account to listen to some music. To do that, we need to give our email address and password. This means that our computer or smartphone needs to send a “POST” request with the given information to Spotify’s server. If we sent the correct data for login, the server would let us into our Spotify account successfully. But during the response, our computer gets a cookie which will help us log in the next time without this whole process. This cookie acts as an ID card; it identifies us immediately.

Telephones and Packets

The telephone system became another catalyst for the creation of the Internet. If the connection path between a telephone call is broken, the connection is cut off. This weakness became a matter of national security in the early 1960s as they feared a Soviet attack which would cripple their communications in effective retaliation. Dr. Paul Baran, who was hired to fix this problem, helped to create Hot Potato Routing, which meant a message transiting a network got to where it was going even if part of the network was destroyed.

This routing led to Packet Switching Packet Switching. The network chops up a document into a lot of pieces (packets). Each packet was like a postcard with a to and from address that would get launched into the network and operate independently. The final destination takes all these packets and puts them back into a long document at the end. This design for a network, a Packet Switching Network, gets its message through better than telephones as long as some part of the network functions.

Bob Taylor’s Frustrations

The first computer networks stem from the frustrations of a single ARPA scientist, Bob Taylor, who needed a separate terminal to log into each of several time-shared computers at research centers across the country. He knew a single terminal would suffice if all the computers were in a common network.

Taylor was given a million dollars to work on this problem where he recruited Larry Roberts, who built the world’s first experimental connection between two computers. Roberts was inspired by a fellow engineer, who suggested building identical special-purpose mini computers, called Interface Messaging Processors ( IMPs ), to form the network and tie each computer to one of them.

He sought out the engineers to build the Advance Research Projects Agency’s computer network, the ARPANET. The plan was to network 4 identical IMPs. Each one would be attached to a different kind of mainframe computer at three different universities across California and one in Utah.

Photo of the very first IMP

BBN and IMPS and Grad Students

A small Cambridge Massachusetts company known as BBN won the contract and went to work building the IMPs for the ARPANET on January 1st, 1969. Both BBN and UCLA were under a tight deadline. Writing the code to have the computers communicate with the IMPs were left to the graduate students or staff members. There was concern with the phone lines, which were built for voice traffic and not data or running the mainframe computers nonstop. By the end of 1969, all four sites were interconnected and had worked the way it was intended.

ARPANET

In April 1971, the ARPANET had 18 mainframe computers hooked into the network. At first there was not much to do beyond transferring data files among its members. Then, a new capability was added that would take the usefulness of computer networking to a new level. Raymond Tomlinson created this killer application called email. Tomlinson gave the world the @ sign.

Internet begun in 1969 as an experimental computer network. Anyone could contribute ideas about its use and how to improve it. Ideas were offered as requests for comments so everyone would free feel to discuss and criticize them. By 1972 the ARPANET had fewer than 25 sites online, yet more were coming. What sparked the increase was the first international conference on computers and communication sponsored by ARPA.

The ARPANET network map in 1974

Networks

Soon other networks emerged. LAN (local area networks) linked computers throughout offices; WAN (wide area networks) linked computers across buildings or campuses. Each new network was like a foreign country speaking its own language and independent of one another. They needed a common protocol for them to communicate.

In 1973, Ben Cerf and Bob Kahn teamed up to develop a set of rules or procedures known as the TCP/IP protocol and if followed by different computer networks, would allow them to pass their message packets back and forth. The Gateway, as Cerf and Kahn named it, knew how to talk to each of the networks it was connected to. In 1983 TCP/IP was adopted as a standard.

Under 1980s technology it was difficult to research a topic. Tim Berners-Lee, a physicist at the CERN Research Institute in Switzerland developed the software to follow threads of knowledge more easily within the information stored throughout the internet. By using a computer mouse to click on a word or phrase linked across the Internet, one could instantaneously follow a link from site to site around the web in pursuit of a topic. The crisscrossing of these threads across the globe reminded him of a web so he named his invention the World Wide Web.

On June 9, 1992, Congress passed a bill taking the Internet out of the exclusive hands of government and into the public. The Internet had crossed the Rubicon.
