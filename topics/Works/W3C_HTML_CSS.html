W3C and HTML and CSS

The Foundations

Sir Tim Berners-Lee as a kid was always really interested in trains; he built some electronic gadgets to control his model railway, but after a while he had a revelation that he liked more electronics than his train cars. So, he went to Oxford University, then he became a software engineer at CERN - large particle physics laboratory near Geneva -. During his work at CERN, he noticed that there was a problem with sharing information among colleagues.

What was his solution? Berners-Lee realized that using the Internet and the newly emerging technology called Hypertext, he could make the knowledge exchange among coworkers or just any researchers much faster and reliable. By October of 1990, Berners-Lee had written three technologies with fundamentally importance that became the foundation of the Web we know today.

  • HTML (Hyper Text Markup Language): It is a formatting language for the Web.
  • URI (Uniform Resource Identifier): This is a unique address that we use for identifying each resource on the web. Nowadays, we call it as a URL.
  • HTTP (Hyper Text Transfer Protocol): It allows for the retrieval of linked resources from across the Web.

Soon after this, the first webpage appeared on the Web, then CERN invited other creators outside of the organization to join their new community on the Internet. It was a very impactful moment when CERN announced in 1993 that the World Wide Web technology would be available for anyone to use on a royalty-free basis. This announcement sparked creativity, collaboration, and innovation world-wide.

W3C

LIn 1994 Berners-Lee left CERN and went to MIT (Massachusetts Institute of Technology) where he found the World Wide Web Consortium (W3C). His goal was to help the Web reach its full potential; to achieve this, the underlying technologies must become global standards, implemented in the same way across the world. So, the international community of W3C became devoted to developing open standards. They produced some revolutionary ideas which work as guidelines today:

  • Decentralization: No permission is needed from a central authority to post anything on the Web.
  • Non-discrimination: It is also known as Net Neutrality. This principle says that if we connect to the Internet with a certain quality of service, and others connect with that or a greater quality of service, then both parties will communicate at the same level.
  • Bottom-up design: Code is being written and controlled by everyone, not just a small group of experts; this encourages maximum participation and experimentation.
  • Universality: All the computers involved in making the Web must speak the same languages to each other. It does not matter whether people use different hardware, where they live, or what cultural and political beliefs they have.
  • Consensus: For universal standards to work, everyone had to agree to use them. Berners-Lee and other contributors achieved this consensus by giving everyone a say in creating the standards through a transparent, participatory process at W3C.

Thanks to the community of W3C, the Web in its form we know enables human communication, commerce, and opportunities to share knowledge. Tim Berners-Lee said in a tweet: “This is for everyone”, and we should keep it that way.

HTML

HTML is the authoring language used for creating documents on the Web. We use it to give instructions to our web browser how to display the website we asked for. These instructions determine for example how the headings and text should look, which image should load, or what other special functions are available. HTML can do this by using the so-called tags that have attributes. For example,<p> means a paragraph break. As we look at the web page, we do not see the HTML; it is hidden from our view; we are able to see only the results.

HTML has evolved a lot throughout the years. With every update, making websites became much easier, efficient, and websites looked always nicer. Let us have a look at the different steps a bit.

  • HTML 1.0: It was released in 1993 with the intention of sharing information that can be readable and accessible via web browsers. The language was limited; it had support for basic elements like text controls and images.
  • HTML 2.0: It was published in 1995; it contained all the features of HTML 1.0 along with few additional features. W3C was already created at this time, so standardization efforts started with this version.
  • HTML 3.2: It was developed in 1997. This version added widely deployed features such as tables, text flow around images, superscripts, and subscripts. Another important feature what HTML 3.2 implemented was support for CSS (Cascading Style Sheets).
  • HTML 4.01: This version came out in 1999; It extended the support of cascading styling sheets. In version 3.2, CSS were embedded in HTML page itself, but in the new version the concept of external styling sheet emerged. Under this concept, an external CSS file could be developed, and this external styling file could be included in HTML itself.
  • HTML 5 This is the latest version of HTML; it became accessible for developers in 2014. It came up with lots of HTML tags support: email, password, audio tag, semantic tags (<header>,<nav>,<footer>), and section tag.

This is an example of a simple HTML code. We have a heading with a paragraph.
<!DOCTYPE html>
<html>
<body>
<h1>My First Heading</h1>
<p>My first paragraph.</p>
</body>
</html>

CSS

Most of us think that HTML and CSS were created simultaneously. But for years after Berners-Lee invented the World Wide Web, there was no such thing as CSS.

Because the layout was not a priority for Berners-Lee, others started to think about how to render elements on a website. A few thinkers emerged: Pei-Yuan Wei, Marc Andreessen and Håkon Wium Lie. Pei-Yuan Wei created a graphical ViolaWWW Browser in which he incorporated his stylesheet language. It did not work properly, but it inspired others. In the meantime, Andreessen had taken a different approach in his browser, Netscape Navigator. He did not make a separate language devoted to styling a website; he simply just extended HTML to include more presentational HTML tags that could be used to design web pages.

Finally, Håkon Wium Lie – a Norwegian web pioneer who worked at CERN with Berners-Lee – came out with the idea of CSS, and it grabbed everyone’s attention immediately. His goal was that using CSS helps people create a uniform look across several pages of a web site. CSS was simple compared to its predecessors. It is a declarative programming language. This means we describe our rules for the HTML document, and after that, we let the browser handle the rendering according to the given rules. At that time, web pages were built by amateurs and hobbyists, so giving CSS a predictable, forgiving format made people acquire this new language much faster.

CSS is special language because it allows for styles to cascade. The cascade feature means that styles can inherit and overwrite other styles that had previously been declared, following a somewhat complicated hierarchy. The biggest breakthrough of all, however, is that it is possible to use multiple stylesheets on the same page.

Here is an example how to insert CSS into HTML directly. Here we gave the background a color and styled the heading too.
<!DOCTYPE html>
<html>
<head>
<style>
body {
 background-color:linen;
 }
h1 {
 color:maroon;
 margin-left:40px;
 }
</style>
</head>
</body>
<h1>My First Heading</h1>
<p>My first paragraph.</p>
</body>
</html>

Importance of CSS

It seems that CSS is not going to be replaced anytime soon due to its immense evolution. As it becomes more and more powerful, its importance is growing too. What are these benefits?

  • It helps web pages load faster.
  • It provides compatibility with multiple browsers.
  • CSS allows the same markup page to come in different display styles.
  • CSS style sheets make it easier for websites to appear in search engine results.
  • When CSS separates the content of our website from its design language, we can significantly reduce the size of our file transfer.
